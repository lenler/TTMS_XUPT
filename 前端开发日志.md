## 2026-05-25：路由骨架 + 布局完善 + 路由守卫 + 管理端登录（feat/frontend-routes）

### 完成内容

| 模块 | 文件 | 说明 |
|------|------|------|
| 管理端路由表 | `routes/admin.routes.tsx` | 17 条路由，包含所有管理模块 |
| 观众端路由表 | `routes/customer.routes.tsx` | 10 条路由，含选座、订单、登录等 |
| AdminLayout | `layouts/AdminLayout.tsx` | 可折叠侧边菜单（6 个一级菜单）、顶栏用户信息、退出按钮、页脚 |
| CustomerLayout | `layouts/CustomerLayout.tsx` | 顶部导航栏（首页/放映/榜单/联系我们）、登录/注册入口、我的订单 |
| AdminGuard | `components/common/AdminGuard.tsx` | 无 token → 跳转 `/admin/login` |
| CustomerGuard | `components/common/CustomerGuard.tsx` | 无 customerToken → 跳转 `/login` |
| 管理端登录页 | `pages/admin/Login.tsx` | 工号+密码表单、表单校验、调用 API、已登录自动跳转 |
| Auth Service | `services/admin/auth.ts` | `adminLogin` / `getCurrentUser` / `getCurrentUserMenus` |
| 登录 Mock | `mocks/handlers/auth.ts` | 成功（admin/123456）、失败（20001）、未登录（20002） |
| Handlers 汇总 | `mocks/handlers/index.ts` | 注册所有模块 handlers |
| App.tsx | `App.tsx` | 管理端+观众端路由组装，Suspense 懒加载全局 Loading |

### 关键设计决策

- **路由方式**：直接 JSX `<Route>` 声明（避免 `RouteObject` spread 的类型兼容问题）
- **登录页独立布局**：`/admin/login` 不走 AdminLayout，只有通过守卫的页面才使用 SaaS 布局
- **观众端 token 独立**：观众端使用 `customerToken`（localStorage 不同 key），与管理端 token 隔离
- **Mock 基准账号**：工号 `admin`，密码 `123456`

### 验收结果

- `pnpm build`（tsc + vite build）通过
- 登录链路：`/admin/login` → 输入 admin/123456 → 跳转 `/admin/dashboard` → 侧边菜单可见
- 路由守卫：直接访问 `/admin/studio` → 跳转 `/admin/login`
- 退出登录：点击退出 → 清除 token → 跳转登录页
