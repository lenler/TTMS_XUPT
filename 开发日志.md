## 2026-05-25 — 接口文档复审与核心链路收紧

### 修改内容

- [x] **新增状态枚举章节**（2.6）：定义票状态（available/locked/sold/checked/refunding/refunded）、订单状态（unpaid/paid/refunding/refunded/cancelled）及状态流转图
- [x] **明确 scheduleId + seatId → ticketId 映射规则**：前端选座位，后端按此规则锁定对应票
- [x] **锁座接口响应增加 ticketId**：锁定成功即返回 ticketId，供后续下单/支付链路使用
- [x] **下单接口简化为仅传 lockToken**：不再重复传 seatIds 或 ticketIds
- [x] **补齐 GET /admin/api/schedules/:id/tickets**：售票员可查询某场演出的票/座位状态
- [x] **修正验票接口**：改为 `POST /admin/api/tickets/:ticketId/verify`，请求体中不再冗余传 ticketId
- [x] **设置座位接口返回标准座位结构**：返回 seats[] + layout，便于前端直接刷新
- [x] **新增演出计划补充副作用说明**：自动基于演出厅座位批量生成 Ticket
- [x] **统一退票规则**：管理端与观众端均遵循相同状态机（票 sold→refunding→refunded→available）

### 审查结论

核心链路（选座 → 锁定 → 下单 → 支付 → 退票）已收紧，接口可作为前后端契约投入 Mock 开发。
