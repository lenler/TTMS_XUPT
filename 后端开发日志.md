### 关键业务规则

- 演出计划创建时，校验演出厅和剧目必须处于 `ACTIVE` 状态。
- 排期冲突按 `[showTime, showTime + durationMinutes)` 时间段进行重叠判断。
- 创建演出计划后，根据演出厅座位自动生成对应演出票。
- 下单时只允许 `AVAILABLE` 票据进入 `LOCKED` 状态，并写入 `lock_time`。
- 支付确认时校验订单状态、支付金额、票据锁定状态和锁票超时时间。
- 支付成功后票据进入 `SOLD`，并清空锁票时间。
- 已验票 `CHECKED` 的票据禁止退票。
- 验票只允许 `SOLD` 状态票据通过，通过后变更为 `CHECKED`。
- 删除类操作采用状态停用，保留历史销售和统计数据。

## 2026-05-26：票据管理模块独立分支（codex/ticket-management）

### 完成内容

| 模块 | 文件/路径 | 说明 |
|------|-----------|------|
| 票据实体 | `ttms/backend/src/main/java/com/hantang/ttms/domain/Ticket.java` | 明确最小字段：`schedule_id`、`seat_id`、`price`、`status`、`lock_time`，并保留乐观锁 `version` |
| 票据状态 | `TicketStatus.java` | 定义 `AVAILABLE`、`LOCKED`、`SOLD`、`CHECKED`、`REFUNDED`、`VOIDED` |
| 票据仓储 | `TicketRepository.java` | 支持按场次查票、按 ID 批量查票、统计票据状态，并加载座位/场次关联数据 |
| 票据接口 | `TicketController.java` | 支持 `GET /tickets`、`GET /tickets/{id}`、`POST /tickets/{id}/check-in` |
| 票据服务 | `TicketService.java` / `TicketServiceImpl.java` | 实现票据查询和入场验票，验票通过后票据由 `SOLD` 变更为 `CHECKED` |
| 票据响应 DTO | `TicketResponse.java` / `TicketMapper.java` | 返回票据 ID、场次 ID、座位 ID、行列、价格、状态、`lockTime` |
| 锁票交易 | `SaleServiceImpl.java` | 下单锁票、过期锁释放、支付确认、退票时维护票据状态和 `lock_time` |
| 表结构约束 | `ttms/database/schema.sql` | `tickets` 表包含 `lock_time`，并添加 `UNIQUE KEY uk_ticket_schedule_seat (schedule_id, seat_id)` |
| 公共响应 | `PageResult.java` | 补充全组共用分页响应结构，和 `ApiResponse` 一起作为后续列表接口约定 |
| 测试配置 | `ttms/backend/src/test/resources/application.yml` | 测试环境使用 H2，避免 Maven 测试依赖本机 MySQL 账号 |

### 关键规则

- `ticket(schedule_id, seat_id)` 必须唯一，防止同一场次同一座位重复出票。
- 下单时只允许 `AVAILABLE` 票据锁定为 `LOCKED`。
- 锁票记录统一使用数据库字段 `lock_time`。
- 支付确认时必须校验票据仍为 `LOCKED` 且 `lock_time` 未超时。
- 支付成功后票据变更为 `SOLD`，并清空 `lock_time`。
- 退票时已验票 `CHECKED` 的票据不可退。
- 验票只允许 `SOLD` 状态票据通过，通过后变更为 `CHECKED`。

### 完成内容

| 模块 | 文件/路径 | 说明 |
|------|-----------|------|
| Maven 依赖 | `ttms/backend/pom.xml` | 将 Spring Data JPA 替换为 `mybatis-spring-boot-starter`，保留 `jakarta.persistence-api` 兼容实体注解 |

| Repository 层 | `repository/*.java` | 将原 JPA Repository 改为 MyBatis 注解 Mapper，并保留 `save/findById/findAll` 等 Service 调用方法 |
| 订单票据适配 | `SaleServiceImpl.java` | 下单锁票后显式保存票据，避免依赖 JPA 脏检查 |
| 测试建表 | `src/test/resources/schema.sql` | 增加 H2/MySQL 兼容建表脚本，保证 `mvn test` 不依赖 JPA 自动建表 |
| 文档 | `ttms/docs/architecture.md` / `ttms/docs/member-b-development.md` | 更新技术栈和持久层说明 |

### 适配说明

- `tickets(schedule_id, seat_id)` 唯一约束保留。
- 票据 `version` 字段保留，MyBatis 更新票据时显式 `version + 1`。
- 销售单保存时显式插入 `sale_items`，不再依赖 JPA 级联。
- 排期、票据、销售查询使用 JOIN 或嵌套查询补齐 DTO 映射需要的剧目、影厅、座位信息。
