# 后端开发日志

## 2026-05-26 演出厅管理模块本地实现

### 本次完成

- 新建 `ttms-backend` Spring Boot 后端工程，技术栈为 Spring Boot 3.3.6、MyBatis XML Mapper、MySQL、JDK 17。
- 新增统一响应结构、分页结构、业务异常和全局异常处理，响应码遵循 `doc/接口设计文档.md`。
- 完成演出厅管理 CRUD 接口：
  - `GET /admin/api/studios`
  - `GET /admin/api/studios/{id}`
  - `POST /admin/api/studios`
  - `PUT /admin/api/studios/{id}`
  - `DELETE /admin/api/studios/{id}`
- `DELETE` 接口按停用处理，将 `status` 更新为 `0`，不物理删除数据。
- 新增 `schema.sql` 和 `data.sql`，用于启动时初始化 `studio` 表和演出厅测试数据。
- 新增 `.gitignore`，避免 Maven `target/` 等构建产物进入版本控制。
