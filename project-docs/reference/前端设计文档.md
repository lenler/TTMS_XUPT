# TTMS 前端设计文档

## 1. 文档说明

本文档为 TTMS 前端工程化设计文档，面向子智能体执行开发任务。文档定义了技术栈、目录结构、路由表、组件规范、接口封装、Mock 策略和分阶段验收标准。

**接口契约**：以 `doc/接口设计文档.md` 为唯一接口契约，所有字段类型、状态枚举、错误码、分页格式必须按接口文档统一封装。

---

## 2. 技术栈与版本

| 类别 | 技术 | 说明 |
|------|------|------|
| 语言 | TypeScript | 严格模式，所有类型必须显式声明 |
| 框架 | React | 函数组件 + Hooks，使用脚手架安装的当前稳定版本 |
| UI 组件库 | Ant Design | 统一使用 antd 原生组件，不引入其他 UI 库 |
| 路由 | React Router | 管理端与观众端分开路由文件 |
| HTTP 客户端 | Axios 1.x | 统一封装拦截器与错误处理 |
| 状态管理 | Zustand | 仅用于 token、用户信息、菜单、购票临时状态等跨页面状态 |
| Mock | MSW (Mock Service Worker) | 浏览器端拦截，无需额外服务 |
| 构建工具 | Vite | 开发服务器 + 生产构建 |
| 包管理 | pnpm | workspace 暂不启用，单包结构 |

> 版本说明：截至当前检查，npm 最新版本分别为 `react@19.2.6`、`antd@6.4.3`、`react-router-dom@7.15.1`、`vite@8.0.14`、`msw@2.14.6`。实际开发时不在文档中强锁小版本，以 `package.json` 和锁文件为准。

### 2.1 样式原则

- 优先使用 Ant Design 原生组件、默认交互和默认视觉规范。
- 页面布局优先使用 Ant Design `Layout`、`Menu`、`Breadcrumb`、`Card`、`Table`、`Form`、`Modal`、`Drawer`、`Tabs`、`Descriptions`、`Statistic` 等组件。
- 颜色、圆角、字号、间距优先使用 Ant Design 主题 token，不单独设计一套视觉系统。
- 不引入 Tailwind CSS、Bootstrap、Material UI、Ant Design Pro 或其他 UI/样式框架。
- 不大面积覆盖 antd 类名，不写复杂全局样式；仅允许少量页面级 CSS 处理布局间距、空状态区域、座位图网格等 antd 无法直接表达的局部样式。
- 管理端保持经典 SaaS 后台风格：左侧菜单、顶部栏、内容区、表格、筛选表单、弹窗表单。避免营销页式大图、复杂渐变和装饰性视觉元素。

## 7. 接口封装规范

### 7.1 Axios 实例

```typescript
// services/request.ts

import axios from 'axios';
import { message } from 'antd';

const request = axios.create({
  baseURL: '',           // Mock 期间用 MSW 拦截，不设真实 baseURL
  timeout: 10000,
});

// 请求拦截器：注入 token
request.interceptors.request.use((config) => {
  const token = useAuthStore.getState().token;
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器：统一错误处理
request.interceptors.response.use(
  (response) => {
    const { resCode, resMsg } = response.data;
    if (resCode !== '10000') {
      message.error(resMsg);
      return Promise.reject(new Error(resMsg));
    }
    return response.data;           // 只返回 { resCode, resMsg, data }，调用方直接拿 data
  },
  (error) => {
    if (error.response?.status === 401) {
      useAuthStore.getState().logout();
    }
    message.error('网络请求失败');
    return Promise.reject(error);
  }
);

export default request;
```

### 7.2 API 函数示例

```typescript
// services/admin/studio.ts

import request from '../request';
import type { ApiResponse, PageData, PageParams } from '@/types/api';
import type { Studio } from '@/types/models';

/** 查询演出厅列表 */
export function getStudios(params: PageParams) {
  return request.get<ApiResponse<PageData<Studio>>>('/admin/api/studios', { params });
}

/** 查询单个演出厅 */
export function getStudioById(id: number) {
  return request.get<ApiResponse<Studio>>(`/admin/api/studios/${id}`);
}

/** 新增演出厅 */
export function createStudio(data: Omit<Studio, 'id' | 'status'>) {
  return request.post<ApiResponse<{ id: number }>>('/admin/api/studios', data);
}

/** 修改演出厅 */
export function updateStudio(id: number, data: Omit<Studio, 'id' | 'status'>) {
  return request.put<ApiResponse<null>>(`/admin/api/studios/${id}`, data);
}

/** 删除演出厅 */
export function deleteStudio(id: number) {
  return request.delete<ApiResponse<null>>(`/admin/api/studios/${id}`);
}
```

### 7.3 调用规范

- 所有 API 函数放在 `services/` 对应模块文件中
- 调用方通过 `import { getStudios } from '@/services/admin/studio'` 使用
- 调用方不直接使用 `request`，必须通过 service 函数
- 每个 service 函数返回 `Promise<ApiResponse<T>>`，调用方取 `.data` 使用

## 8. 通用组件规范

### 8.1 PageTable（通用分页表格）

封装 Ant Design `Table`，统一处理：

- 分页参数（`page`, `pageSize`, `total`）
- 搜索关键词联动
- 操作列按钮（编辑、删除）
- 删除二次确认

```typescript
// 使用示例
<PageTable<Studio>
  columns={columns}
  fetchData={(params) => getStudios(params)}
  onEdit={(record) => navigate(`/admin/studio/${record.id}`)}
  onDelete={(id) => deleteStudio(id)}
  rowKey="id"
/>
```

### 8.2 useCRUD（通用 CRUD Hook）

提供列表页标准能力：`list`, `loading`, `pagination`, `keyword`, `setKeyword`, `refresh`。

```typescript
// hooks/useCRUD.ts
function useCRUD<T>(fetchFn: (params: PageParams) => Promise<ApiResponse<PageData<T>>>) {
  // 返回 { list, loading, pagination, keyword, setKeyword, refresh }
}
```

### 8.3 表单规范

- 新增/修改统一使用 Ant Design `Form` + `Modal` 或独立 `Detail` 页面
- 表单校验规则使用 Ant Design `Form.Item` 的 `rules` 属性
- 提交时调用对应 service 函数，成功后 `message.success` + 返回列表刷新
- 日期使用 `DatePicker`，金额使用 `InputNumber` 精度 `precision={2}`
- 下拉选择（剧目类型、语种、演出厅等）通过字典接口或关联查询获取

## 9. Mock 策略

### 9.1 MSW 配置

- 开发环境自动启用（`VITE_ENABLE_MOCK=true`），生产构建自动排除
- Mock 数据放在 `src/mocks/data/*.json`
- Handler 按模块拆分，汇总在 `src/mocks/handlers/index.ts`

### 9.2 Mock 数据要求

- 每条接口至少覆盖正常响应（`resCode: "10000"`）和一种异常响应
- 列表接口需根据 `page`/`pageSize` 参数正确分页
- POST/PUT/DELETE 操作需模拟数据变更效果（在内存中修改 `Map` 或直接修改静态数据引用）
- 登录 Mock 需返回固定 token，后续请求校验该 token
- 票状态流转（`available → locked → sold → checked`）需在 Mock 中体现基础路径

### 9.3 Mock 示例

```typescript
// mocks/handlers/studio.ts
import { http, HttpResponse } from 'msw';
import studiosData from '../data/studios.json';

let studios = [...studiosData];

export const studioHandlers = [
  // 查询列表
  http.get('/admin/api/studios', ({ request }) => {
    const url = new URL(request.url);
    const keyword = url.searchParams.get('keyword') || '';
    const page = Number(url.searchParams.get('page')) || 1;
    const pageSize = Number(url.searchParams.get('pageSize')) || 10;

    let filtered = studios;
    if (keyword) {
      filtered = studios.filter((s) => s.name.includes(keyword));
    }

    const start = (page - 1) * pageSize;
    const list = filtered.slice(start, start + pageSize);

    return HttpResponse.json({
      resCode: '10000',
      resMsg: '请求成功',
      data: { list, total: filtered.length, page, pageSize },
    });
  }),

  // 新增
  http.post('/admin/api/studios', async ({ request }) => {
    const body = await request.json();
    const newStudio = { ...body, id: Date.now(), status: 1 };
    studios.push(newStudio);
    return HttpResponse.json({
      resCode: '10000',
      resMsg: '添加成功',
      data: { id: newStudio.id },
    });
  }),
];
```

## 12. 子智能体执行规范

### 12.1 开发前检查清单

- [ ] 阅读 `doc/接口设计文档.md` 中与当前模块相关的接口定义
- [ ] 阅读本文档中与当前模块相关的章节
- [ ] 确认当前阶段的前置模块已完成

### 12.2 每个模块的交付物

1. `pages/` — 页面组件（列表页 + 详情页）
2. `services/` — API service 函数（如需新接口）
3. `mocks/handlers/` — Mock handler（含正常 + 异常响应）
4. `mocks/data/` — Mock 静态数据（如需）

### 12.3 代码规范

- 所有组件使用函数组件 + TypeScript
- Props 类型必须显式声明（interface 或 type）
- 每个函数和每个类写注释（中文）
- 使用 Ant Design 提供的原生组件，不自己造轮子
- 样式尽量使用 antd 默认样式和主题 token，不做大面积二次封装或视觉重写
- 表单/表格尽量复用 `PageTable`、`useCRUD` 等通用组件/Hook
- 状态管理：跨页面共享状态用 Zustand，页面内局部状态用 `useState`/`useReducer`

### 12.4 提交规范

- 每个阶段结束后提交一次，commit message 格式：`feat(模块名): 完成XX功能`
- 如：`feat(studio): 完成演出厅管理 CRUD 与座位设置`
